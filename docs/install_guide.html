▸ BEFORE YOU START
Install order matters here. MongoDB must be installed before Open5GS. ZMQ must be installed before srsRAN is compiled. Java must be installed before Keycloak. Follow the sections in order top to bottom. All commands assume you are running Ubuntu 24.04 LTS on x86_64 with sudo access.
01 System Prerequisites APT
Run these first. Installs core build tools, networking utilities, and common dependencies needed by everything else in this guide.
Update & install base packages
# Update package index and upgrade existing packages
sudo apt update && sudo apt upgrade -y

# Install core build tools and utilities
sudo apt install -y \
  build-essential cmake gcc g++ git curl wget gnupg \
  pkg-config software-properties-common \
  net-tools iproute2 tshark wireshark-common
Enable IP forwarding (required by Open5GS UPF)
# Enable now
sudo sysctl -w net.ipv4.ip_forward=1

# Make it persistent across reboots
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
Disable UFW firewall (required by Open5GS for lab use)
Open5GS uses multiple loopback addresses that UFW blocks by default. Disable for lab purposes.
sudo ufw disable
02 MongoDB APT
Open5GS uses MongoDB as its subscriber database (NRF, PCF, UDR). Must be installed before Open5GS. Ubuntu 24.04 (noble) does not have MongoDB in its default repos — you add it from MongoDB's own repo. The jammy (22.04) repo is used because MongoDB has not yet published a noble-specific build, but it works fine on 24.04.
Add MongoDB 8.0 repository & install
# Import MongoDB public GPG key
sudo apt install -y gnupg
curl -fsSL https://pgp.mongodb.com/server-8.0.asc | \
  sudo gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor

# Add repo (using jammy — works on noble/24.04)
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] \
https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/8.0 multiverse" | \
  sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list

# Install
sudo apt update
sudo apt install -y mongodb-org

# Start and enable on boot
sudo systemctl start mongod
sudo systemctl enable mongod
▸ VERIFY
sudo systemctl status mongod
Should show Active: active (running)
03 Open5GS (5G Core) PPA
Full open-source 5G SA core — runs AMF, UPF, SMF, NRF, and all other 5G core network functions. Install via the official Launchpad PPA, which supports Ubuntu 24.04 (noble). Also installs the WebUI for subscriber management.
Install Open5GS via PPA
# Add the official Open5GS PPA
sudo add-apt-repository ppa:open5gs/latest
sudo apt update
sudo apt install -y open5gs
Install WebUI (subscriber management)
Requires Node.js. The WebUI lets you register virtual UE subscribers in MongoDB via a browser at http://localhost:9999.
# Install Node.js (LTS) via NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Install the WebUI
curl -fsSL https://open5gs.org/open5gs/assets/webui/install | sudo -E bash -
Add TUN interface for UPF
Required for the UPF to route data plane traffic. Run once after install.
# Create ogstun interface (UPF data plane)
sudo ip tuntap add name ogstun mode tun
sudo ip addr add 10.45.0.1/16 dev ogstun
sudo ip addr add 2001:db8:cafe::1/48 dev ogstun
sudo ip link set ogstun up

# NAT rule so UE traffic can reach internet/loopback
sudo iptables -t nat -A POSTROUTING -s 10.45.0.0/16 ! -o ogstun -j MASQUERADE
Start all Open5GS services
# All core NFs start automatically after install.
# Check they are running:
sudo systemctl status open5gs-amfd
sudo systemctl status open5gs-upfd
sudo systemctl status open5gs-smfd

# If any are stopped, start manually:
sudo systemctl start open5gs-amfd open5gs-upfd open5gs-smfd \
  open5gs-nrfd open5gs-ausfd open5gs-udmd open5gs-udrd \
  open5gs-pcfd open5gs-nssfd open5gs-bsfd
▸ VERIFY
Open browser → http://localhost:9999
Login: admin / 1423
You should see the subscriber management dashboard.
▸ NOTE
Default configs bind all NFs to loopback addresses (127.0.0.x). This is correct for a single-machine lab setup — no changes needed yet. You will edit amf.yaml and upf.yaml later when configuring for srsRAN.
04 srsRAN Project + ZeroMQ (Virtual RAN) BUILD FROM SOURCE
srsRAN Project is the 5G NR gNodeB software stack. For Setup A, it runs in ZMQ mode — no radio hardware needed. ZMQ replaces the RF frontend with TCP sockets, allowing a virtual gNB to connect to Open5GS over the N2/N3 interfaces. Must be compiled from source with ZMQ enabled. Build takes 5–15 minutes depending on your machine. Requires at least 4 CPU cores and 4 GB RAM to run.
Step 1 — Install build dependencies
sudo apt-get install -y \
  cmake make gcc g++ pkg-config \
  libfftw3-dev libmbedtls-dev libsctp-dev \
  libyaml-cpp-dev libgtest-dev
Step 2 — Install ZeroMQ
ZMQ must be installed before srsRAN is compiled. If you compile srsRAN first and then add ZMQ, you must delete the build directory and recompile.
# Install ZMQ dev package from apt
sudo apt-get install -y libzmq3-dev
Step 3 — Clone and build srsRAN Project
The -DENABLE_ZEROMQ=ON flag is critical. Watch the cmake output and confirm it detects ZMQ (see verify box below).
# Clone the repository
git clone https://github.com/srsran/srsRAN_Project.git
cd srsRAN_Project
mkdir build && cd build

# Configure with ZMQ enabled
cmake ../ -DENABLE_EXPORT=ON -DENABLE_ZEROMQ=ON

# Build (uses all available CPU cores)
make -j$(nproc)

# Optional: install system-wide
sudo make install
▸ VERIFY ZMQ WAS DETECTED DURING CMAKE
In the cmake output, look for this line:
-- Found libZEROMQ: /usr/local/include, /usr/local/lib/libzmq.so
If it says No package 'ZeroMQ' found, ZMQ was not installed before cmake ran — delete the build dir, reinstall libzmq3-dev, and rerun cmake.
▸ NOTE ON CPU CORES
When running the ZMQ gNB (not just compiling), srsRAN Project requires 5+ CPU cores to successfully start up and accept UE connections. This is a runtime constraint, not a compile constraint.
05 Python ML Stack (PLA Classifier) PIP
Python 3 is already on Ubuntu 24.04. This step installs the scientific/ML libraries used for the PLA feature extraction and classifier pipeline: scikit-learn for the SVM/RF classifier, numpy/scipy for signal processing, and optionally TensorFlow or PyTorch if you go the deep learning route.
Install pip and core scientific stack
# Ensure pip is current
sudo apt install -y python3-pip python3-venv

# Create a virtual environment (keeps things clean)
python3 -m venv ~/pla-env
source ~/pla-env/bin/activate

# Install ML and signal processing libraries
pip install numpy scipy scikit-learn matplotlib pandas joblib
Optional — PyTorch (deep learning path)
Only install this if you plan to use an LSTM or autoencoder-based PLA classifier per Melki et al. Not needed for SVM/RF.
# CPU-only PyTorch (no GPU required)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
Install requests library
Used to push PLA trust scores to OPA's REST API.
pip install requests
▸ VERIFY
source ~/pla-env/bin/activate
python3 -c "import sklearn, numpy, scipy; print('OK')"
Should print OK with no errors.
06 Open Policy Agent (OPA) SINGLE BINARY
OPA is a single static binary — no package manager, no dependencies, no installation wizard. Download, make executable, move to PATH. It runs as a REST server on port 8181 and evaluates Rego policies when queried. This is the ZTA policy engine that will receive PLA trust scores from your Python classifier and make permit/deny decisions.
Download and install OPA binary
# Download latest static binary for linux/amd64
curl -L -o opa https://openpolicyagent.org/downloads/latest/opa_linux_amd64_static

# Make executable and move to PATH
chmod +x opa
sudo mv opa /usr/local/bin/
Test OPA in server mode
Run this to confirm OPA starts correctly. It will listen on port 8181. Press Ctrl+C to stop — you don't need it running persistently yet.
# Start OPA server (test run — Ctrl+C to stop)
opa run --server
▸ VERIFY
In a second terminal: 
opa version
curl http://localhost:8181/v1/data
opa version prints the version string. The curl should return {} (empty data store — correct).
07 Keycloak (Identity Provider) REQUIRES JAVA 21
Keycloak is the identity and token issuance component of the ZTA stack. It handles OAuth2/OIDC token issuance for UE identities — OPA then validates these tokens alongside PLA trust scores. Keycloak requires Java 21+. The install is a tarball extract + run, no system-wide package manager needed.
Step 1 — Install Java 21
# Install OpenJDK 21 (current LTS, required by Keycloak 26.x)
sudo apt install -y openjdk-21-jdk

# Set JAVA_HOME
echo 'JAVA_HOME="/usr/lib/jvm/java-21-openjdk-amd64"' | sudo tee -a /etc/environment
source /etc/environment
Step 2 — Download and extract Keycloak
Keycloak 26.x is the current release as of 2025. Check https://github.com/keycloak/keycloak/releases and replace the version number below if a newer release is available.
# Download Keycloak 26.x (check GitHub releases for latest)
cd /opt
sudo wget https://github.com/keycloak/keycloak/releases/download/26.0.7/keycloak-26.0.7.tar.gz
sudo tar -xvzf keycloak-26.0.7.tar.gz
sudo mv keycloak-26.0.7 keycloak
sudo rm keycloak-26.0.7.tar.gz

# Create a dedicated system user
sudo useradd -r -d /opt/keycloak -s /sbin/nologin keycloak
sudo chown -R keycloak:keycloak /opt/keycloak
Step 3 — Start Keycloak in dev mode
Dev mode is appropriate for a research testbed. It uses an embedded H2 database (no PostgreSQL setup needed) and runs on HTTP port 8080. Do not use dev mode for anything internet-facing.
# Start in development mode (runs on http://localhost:8080)
sudo -u keycloak /opt/keycloak/bin/kc.sh start-dev
First startup takes ~30–60 seconds. Leave this terminal open or set up a systemd service (see note below).
Step 4 — Create admin user (first run only)
On first startup, Keycloak will prompt you to create an admin account when you visit the web UI, or you can set it via environment variables before starting:
# Set admin credentials before first start (run before kc.sh)
export KEYCLOAK_ADMIN=admin
export KEYCLOAK_ADMIN_PASSWORD=changeme

# Then start:
sudo -E -u keycloak /opt/keycloak/bin/kc.sh start-dev
▸ VERIFY
Open browser → http://localhost:8080
You should see the Keycloak welcome page. Click Administration Console and log in with your admin credentials.
▸ PORT CONFLICT NOTE
Keycloak defaults to port 8080. Open5GS WebUI also uses port 9999. No conflict there, but if you have anything else on 8080, Keycloak will fail to start. Check with sudo ss -tlnp | grep 8080 first.
08 Prometheus + Grafana (Monitoring) APT
Prometheus scrapes and stores time-series metrics. Grafana visualizes them on dashboards. Together they give you the real-time monitoring panel equivalent to Yoon et al.'s Nokia monitoring interface — but entirely open-source. Prometheus is available directly from Ubuntu 24.04's apt repos. Grafana requires adding its own APT repo.
Install Prometheus & Node Exporter
# Both are in Ubuntu 24.04 apt repos
sudo apt install -y prometheus prometheus-node-exporter

# Enable and start
sudo systemctl enable prometheus prometheus-node-exporter
sudo systemctl start prometheus prometheus-node-exporter
Prometheus UI available at http://localhost:9090 after start.
Add Grafana APT repository and install
# Install dependencies
sudo apt install -y apt-transport-https software-properties-common wget

# Add Grafana GPG key
sudo mkdir -p /etc/apt/keyrings/
wget -q -O - https://apt.grafana.com/gpg.key | \
  gpg --dearmor | \
  sudo tee /etc/apt/keyrings/grafana.gpg > /dev/null

# Add Grafana stable repo
echo "deb [signed-by=/etc/apt/keyrings/grafana.gpg] \
https://apt.grafana.com stable main" | \
  sudo tee /etc/apt/sources.list.d/grafana.list

# Install
sudo apt update
sudo apt install -y grafana

# Enable and start
sudo systemctl enable grafana-server
sudo systemctl start grafana-server
Open firewall ports (if UFW is re-enabled later)
# Only needed if you later re-enable ufw
sudo ufw allow 9090/tcp   # Prometheus
sudo ufw allow 3000/tcp   # Grafana
sudo ufw allow 9100/tcp   # Node Exporter
▸ VERIFY
Prometheus UI → http://localhost:9090
Grafana UI → http://localhost:3000
Grafana default login: admin / admin (you will be prompted to change password on first login).

In Grafana: Connections → Data Sources → Add → Prometheus → URL: http://localhost:9090 → Save & Test.
