▸ IMPORTANT — ORDER MATTERS
Always start components in the exact order listed below. The core network must be up before the RAN connects, the RAN must be up before the UE attaches, and the PLA exporter must be running before Prometheus can scrape data. Starting out of order will cause connection failures that require restarting multiple components.
▸ STARTUP SEQUENCE RUN IN ORDER
01 Verify Open5GS Core Services BACKGROUND Phase 1
Open5GS services auto-start on boot. Verify they are running before proceeding. If any show inactive, run the start command below.
# Check AMF and UPF status
sudo systemctl status open5gs-amfd
sudo systemctl status open5gs-upfd

# If any are stopped, start all core NFs
sudo systemctl start open5gs-amfd open5gs-upfd open5gs-smfd \
  open5gs-nrfd open5gs-ausfd open5gs-udmd open5gs-udrd \
  open5gs-pcfd open5gs-nssfd open5gs-bsfd
▸ VERIFY
Both AMF and UPF show Active: active (running)
02 Restore ogstun Interface ANY TERMINAL Phase 1
The ogstun TUN interface does not persist across reboots. Must be recreated every session. The 2>/dev/null suppresses errors if it already exists.
# Recreate ogstun interface for UPF data plane
sudo ip tuntap add name ogstun mode tun 2>/dev/null
sudo ip addr add 10.45.0.1/16 dev ogstun 2>/dev/null
sudo ip link set ogstun up
sudo iptables -t nat -A POSTROUTING -s 10.45.0.0/16 ! -o ogstun -j MASQUERADE
▸ VERIFY
ip addr show ogstun shows 10.45.0.1/16 assigned and state UP
03 Verify Prometheus & Grafana BACKGROUND Phase 8
Both run as system services and persist across reboots. Verify they are up before starting the pipeline so monitoring is ready when data flows.
# Check status
sudo systemctl status prometheus
sudo systemctl status grafana-server

# If stopped, start them
sudo systemctl start prometheus grafana-server
▸ VERIFY
Prometheus UI → http://localhost:9091
Grafana UI → http://localhost:3000
04 Start OPA Policy Engine ANY TERMINAL Phase 5
OPA does not run as a system service — start it manually. Run in the background with & or in a dedicated terminal. Listens on port 8181.
# Start OPA server in background
opa run --server --log-level info &

# Reload the ZTA policy
curl -X PUT http://localhost:8181/v1/policies/zta_access \
  -H "Content-Type: text/plain" \
  --data-binary @/home/$USER/opa_policies/zta_policy.rego
▸ VERIFY
curl returns {} on policy load — empty object means success
05 Start Keycloak Identity Provider TERMINAL 4 Phase 6
Keycloak runs in dev mode. Open a dedicated terminal and leave it running — it logs to this terminal. First startup takes ~30–60 seconds.
# Start Keycloak in dev mode
export KEYCLOAK_ADMIN=admin
export KEYCLOAK_ADMIN_PASSWORD=changeme
sudo -E -u keycloak /opt/keycloak/bin/kc.sh start-dev
▸ VERIFY
Browser → http://localhost:8080 — Keycloak welcome page loads.
Log in to Administration Console and confirm private5G realm and four UE users exist.
06 Start srsRAN Project gNB TERMINAL 1 Phase 2
The srsRAN Project gNB connects to the Open5GS AMF over the N2 interface. Open a dedicated terminal and leave it running.
# Navigate to build directory and start gNB
cd ~/srsRAN_Project/build
sudo ./apps/gnb/gnb -c /etc/srsran/gnb_zmq.yaml
▸ VERIFY
Output shows: N2: Connection to AMF on 127.0.0.5:38412 completed
Followed by: ==== gNB started ===
07 Start UERANSIM gNB TERMINAL 2 Phase 3
UERANSIM's gNB simulator connects to the same Open5GS AMF and handles the UE attach procedure. Open a second dedicated terminal.
# Start UERANSIM gNB
cd ~/UERANSIM
sudo build/nr-gnb -c config/gnb.yaml
▸ VERIFY
Output shows: NG Setup procedure is successful
08 Start UERANSIM UE TERMINAL 3 Phase 3
The simulated UE attaches to the core network through UERANSIM's gNB, authenticates via 5G AKA, and establishes a PDU session. Open a third dedicated terminal.
# Start UERANSIM UE
cd ~/UERANSIM
sudo build/nr-ue -c config/ue.yaml
▸ VERIFY
Output shows: Initial Registration is successful
Followed by: Connection setup for PDU session[1] is successful, TUN interface[uesimtun0, 10.45.0.2] is up
Open5GS WebUI at http://localhost:9999 shows subscriber as SERVICE_GRANTED
09 Start PLA Prometheus Exporter TERMINAL 5 Phase 8
The PLA exporter computes trust scores from the trained SVM model and exposes them to Prometheus every 15 seconds. Activate the Python virtual environment first.
# Activate venv and start exporter
source ~/pla-env/bin/activate
python3 ~/pla_data/pla_exporter.py
▸ VERIFY
Output shows: PLA Prometheus exporter running on port 9200
Prometheus targets at http://localhost:9091/targets shows pla_pipeline UP
10 Generate MATLAB Features (if retraining) WINDOWS — MATLAB Phase 4
Only required if regenerating signal data or retraining the SVM. If the existing features.csv and saved model files are still present from the last session, skip this step — the exporter and bridge use the saved model directly.
% In MATLAB on Windows — only run if retraining
% 1. Open generate_pla_features.m
% 2. Run script — writes features.csv to WSL2 shared path
% 3. Then in Ubuntu terminal:
source ~/pla-env/bin/activate
python3 ~/pla_data/pla_classifier.py
▸ VERIFY
Python classifier prints classification report and saves updated svm_model.pkl and scaler.pkl to ~/pla_data/
▸ RUNNING THE PIPELINE ON DEMAND
— Run ZTA Bridge (Full End-to-End Evaluation) ANY TERMINAL Phase 7
Run the bridge script to execute a full ZTA evaluation for all four devices. Each run evaluates PLA trust scores → Keycloak identity → OPA policy decision and logs results to the audit log. Run this as many times as needed.
# Run full pipeline evaluation
source ~/pla-env/bin/activate
python3 ~/pla_data/zta_bridge.py

Device 1 (Legitimate) → Trust ~0.79 → UCA_GRANT
Device 2 (Legitimate) → Trust ~0.52 → UCA_DENY (unauthorized slice)
Device 3 (Rogue)     → Trust ~0.31 → TSA_DENY
Device 4 (Rogue)     → Trust ~0.13 → TSA_DENY

▸ SHUTDOWN SEQUENCE REVERSE ORDER
▸ SHUTDOWN ORDER
Always shut down in reverse order — UE first, then RAN, then core-adjacent services. Shutting down the core before the RAN disconnects cleanly can leave stale session state in Open5GS that causes issues on next startup.
01 Stop PLA Exporter TERMINAL 5 
Ctrl+C
02 Stop UERANSIM UE TERMINAL 3 
Ctrl+C
03 Stop UERANSIM gNB TERMINAL 2 
Ctrl+C
04 Stop srsRAN Project gNB TERMINAL 1 
Ctrl+C
05 Stop Keycloak TERMINAL 4 
Ctrl+C
06 Stop OPA ANY TERMINAL 
# If OPA was started with & (background)
pkill opa

# If OPA was started in a dedicated terminal
Ctrl+C
07 Open5GS, Prometheus, Grafana BACKGROUND
These run as system services — they do not need to be stopped between sessions. Leave them running. They will persist across reboots automatically.
